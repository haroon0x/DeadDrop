# Support

Use GitHub Issues for reproducible bugs and GitHub Discussions for setup help, design questions, and usage ideas.

Before filing a bug, check the latest release, read the deployment and architecture guides, and run the relevant health or verification commands.

Include:

- server and worker versions
- operating system and deployment method
- agent mode
- expected and actual behavior
- sanitized logs
- whether the job was queued, running, completed, failed, or cancelled

Never post owner tokens, worker tokens, database URLs, private prompts, repository contents, or sensitive patches.

This is a community open-source project. Support is best-effort and no response-time guarantee is provided. Security reports belong in the private process described by [SECURITY.md](SECURITY.md).
