Documentation

From zero to a verified local-agent queue.

Run the server, configure one trusted Git workspace, and make the worker prove what changed.

Requirements Start the server Build the worker Create a manifest Run the worker Read receipts Go deeper
01

Requirements

You need Docker with Compose, Git, and either a released worker binary or Go 1.22+. Gemini CLI is only required for Gemini mode.

02

Start a durable server

The Compose stack runs FastAPI and PostgreSQL. Secrets stay in your shell environment; database state stays in a named volume.

export OWNER_TOKEN="$(openssl rand -hex 32)"
export WORKER_TOKEN="$(openssl rand -hex 32)"
export POSTGRES_PASSWORD="$(openssl rand -hex 32)"
docker compose up -d --build

Open http://localhost:8000/login and enter OWNER_TOKEN.

03

Build or download the worker

cd worker
go build -o deaddrop-worker .
./deaddrop-worker version

Tagged releases provide Linux, macOS, and Windows binaries with SHA-256 checksums.

04

Create the trust boundary

The manifest is local. The server receives the alias default, not this absolute path.

./deaddrop-worker init \
  --repo /absolute/path/to/project \
  --verify "go test ./..."

The workspace must be a Git root or committed subdirectory. Repeat --verify to add more trusted checks.

05

Run the worker

./deaddrop-worker run \
  --server http://localhost:8000 \
  --token "$WORKER_TOKEN" \
  --manifest deaddrop.manifest.json \
  --agent gemini

Use --agent mock for the bundled deterministic demo, or --agent custom --command-template 'agent "{{prompt}}"' for another CLI.

06

Know what the receipt means

Summary

The agent's human-readable account.

Changed files

Derived from the baseline-relative Git diff.

Verification

Commands and exit states observed by the worker.

Patch

A scoped binary diff; never auto-applied.
